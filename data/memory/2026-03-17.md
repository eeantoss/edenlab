### EdenLab 线上问题排查
- 远程站点 `https://edenlab-nine.vercel.app/workspace` 确认来自仓库 `https://github.com/eeantoss/edenlab`。
- 已提交修复 `0bf93da fix: restore yesterday memo rendering on workspace`，用于恢复 workspace 的“昨日小记”渲染链路。
- 进一步排查发现：EdenLab 线上状态链路被拆成了两套：
  - `/status` 是基于 GitHub Gist 的真实状态源；
  - `/api/star-office-status` 与 `/api/star-office-set-state` 仍是 Vercel 展示模式 stub，导致线上状态更新不同步。
- `public/star-office/index.html` 内部办公室场景实际使用的是 `/status` 与 `/api/set_state`，而外层 workspace 页用的是 `/api/star-office-*`，两者未统一。
- “搬新家 / 找中介”等功能线上报 `unknown error` 的直接原因是接口返回结构不匹配：前端要求 `ok / async / task_id` 和轮询接口 `/assets/generate-rpg-background/poll`，但当前后端仍是占位实现。
- EdenLab 当前多条 assets 接口（如 list / upload / positions / defaults）也是占位返回，字段名与前端预期不一致，导致线上多个功能不可用。
