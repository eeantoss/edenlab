## EdenLab + Star-Office-UI 项目集成

### 背景问题
- 打开 https://edenlab-nine.vercel.app/workspace 时报错：`127.0.0.1 refused connection`
- 控制台发现请求：`http://127.0.0.1:18991/`
- 问题：前端硬编码了本地地址，部署到 Vercel 后无法访问

### 解决方案
1. **修改 workspace 页面**：
   - 从 `iframe src="http://127.0.0.1:18991/"` 改为相对路径
   - 使用环境变量 `NEXT_PUBLIC_STAR_OFFICE_URL`

2. **集成 Star-Office-UI 到 EdenLab**：
   - 复制前端文件到 `public/star-office/`
   - 创建 Next.js API 路由 `/api/star` 替代 Flask backend
   - 修改所有 API 调用路径以适配新的 API 结构

3. **API 路径映射**：
   - `/status` → `/api/star`
   - `/agents` → `/api/star?action=agents`
   - `/set_state` → `/api/star?action=set_state`
   - `/yesterday-memo` → `/api/star?action=yesterday-memo`
